Manual Testing: TutorialsNinja Web Application
1. Project Overview
This project demonstrates manual testing of the TutorialsNinja Demo e-commerce website.
It covers critical workflows including:
•	User login/logout
•	User registration
•	Forgotten password
•	Shopping cart
•	Checkout
The project highlights professional test cases, execution, and bug reporting skills.

2. Tools Used
•	MS Word – bug tracking and reporting
•	Excel – test cases and test scenarios
•	Browsers: Chrome, Microsoft Edge, Firefox
•	Windows: Windows 11

3. Test Scenarios Covered

Login / Logout
o	Valid credentials
o	Invalid credentials
o	Edge cases (special characters, empty fields)

Shopping Cart
o	Add products to cart
o	Remove items from cart
o	Update quantities

User Registration
o	Create new accounts
o	Validate required fields
o	Error handling

5. Project Structure
The project is organized as follows:

TutorialsNinja-Demo-WebApp/
│
├── Test Plan/
│   ├── TestPlan.docx
│   ├── TestScenarios.xlsx
│   └── TestCases.xlsx
│
├── Defects Report/
│   ├── DefectsReport.docx (with screenshots)
│
├── Test Report/
│   ├── TestReport.docx
└── README.md   --> This file

Explanation:
•	Test Plan/ – Contains the overall test plan, scenarios, and detailed test cases. 
•	Defects Report/ – Stores defect logs and screenshots captured during testing. 
•	Test Report/ – Stores completed test reports and defect summaries.

5. Bug Reporting Process
•	Bugs are reported in MS Word with:
o	Bug ID & Description
o	Steps to reproduce
o	Severity and priority
o	Screenshot evidence
